## Conceptos aprendidos

Durante la realización de los ejercicios en Learn Git Branching, se reforzaron los siguientes conceptos:

### 🔹 Commits
Representan puntos en el historial del proyecto. Cada commit guarda los cambios realizados en el repositorio.

### 🔹 Branches
Permiten trabajar en diferentes versiones del proyecto de forma paralela sin afectar la rama principal.

### 🔹 Merge
Se utiliza para unir los cambios de una rama con otra.

### 🔹 Rebase
Permite reescribir el historial de commits, moviendo una rama a otra base para mantener un historial más limpio.

### 🔹 Navegación en el historial
Se aprendió a moverse entre commits usando referencias como `HEAD`, `HEAD~1`, etc.

## Reflexión final

Git es una herramienta fundamental en proyectos reales debido a que:

- Permite llevar un historial completo de cambios.  
- Facilita el trabajo en equipo sin sobrescribir el trabajo de otros.  
- Permite experimentar sin miedo gracias a las ramas.  
- Hace posible revertir errores fácilmente.  
- Es ampliamente utilizado en la industria del software.

En conclusión, Git es esencial para el desarrollo moderno, especialmente en proyectos grandes y que se incluyan a muchas personas trabajando al mismo tiempo.
